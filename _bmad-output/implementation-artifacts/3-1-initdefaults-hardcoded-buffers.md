## Dev Notes

### What InitDefaults IS — and what it is NOT

`InitDefaults` is the **hardcoded fallback** for PATCH and MASTER init templates when user `PatchInit.syx` / `MasterInit.syx` files are absent (D-034). It holds two **packed** byte arrays — the same shape `PatchModel` and `MasterModel` use — ready for `SysExEncoder`, mappers, or `loadFrom()`.

It must **NOT** in this story:
- Load files from disk (`InitTemplateLoader` — Story 3.2)
- Wire into `PluginProcessor` or `ActionDispatcher` (Stories 3.2, 7.2)
- Implement Matrix Mod init **actions** (Story 3.3 — but patch buffer must have cleared mod matrix for consistency)
- Show footer messages or modals (Story 3.2)
- Duplicate descriptor offset tables — consumers use `PatchModel`/`MasterModel` for semantic access

[Source: architecture.md AD-2 service map; D-034 in `.decision-log.md`]

### Open spike: owner-supplied patch values (PRD §9 #4)

PRD explicitly marks INIT hardcoded patch values as **non-blocking**. Guillaume has not yet committed a reference `PatchInit.syx`. The epic AC allows *"documented placeholders pending owner input"*.

**Master path is unblocked:** `Master 1.syx` is the only committed master dump and represents factory-default global parameters (Master Tune/Transpose = 0, all group enables on). Story 1.2 validated it. Reuse those 172 bytes.

**Patch path requires honesty:**
- Do **not** invent musically meaningful module defaults byte-by-byte without a hardware capture.
- Do **not** reuse arbitrary patches (`Patch 5`, `66`, etc.) as init — those are real programs, not factory INIT.
- **Do** ship a structurally valid placeholder (encodable SysEx, cleared Matrix Mod) and flag it for replacement when owner drops `PatchInit.syx` into `Tests/Fixtures/Init/`.

### Matrix Mod bytes in patch init buffer (Story 3.3 alignment)

Matrix Mod packed layout (bytes 104–133, 10 buses × 3 bytes):

| Bus | Source offset | Amount offset | Destination offset |
|---|---|---|---|
| 0 | 104 | 105 | 106 |
| n | 104 + 3n | 105 + 3n | 106 + 3n |

Init state per bus: SOURCE choice index **0** (NONE), AMOUNT **0**, DESTINATION choice index **0** (NONE). [Source: 2-6-matrix-mod-bus-parameter-sysex.md; `PluginDescriptorsMatrixModulation.cpp`]

Even though Story 3.3 sends 0x0B SysEx for Matrix Mod init (not full-buffer replace), the patch init buffer should still carry a cleared matrix so full-patch INIT (Story 3.2 → `PatchModel::loadFrom`) is coherent.

### Recommended API shape

```cpp
// Source/Core/Init/InitDefaults.h
namespace Core
{
    class InitDefaults
    {
    public:
        static constexpr size_t kPatchSize = SysExConstants::kPatchPackedDataSize;
        static constexpr size_t kMasterSize = SysExConstants::kMasterPackedDataSize;
        static constexpr bool kPatchDefaultsArePlaceholder = true; // flip false when owner fixture lands

        static const juce::uint8* patchData() noexcept;
        static const juce::uint8* masterData() noexcept;

        InitDefaults() = delete;
    };
}
```

Copy into models:

```cpp
patchModel.loadFrom(InitDefaults::patchData());
masterModel.loadFrom(InitDefaults::masterData());
```

Story 3.2 `InitTemplateLoader` pattern: try file → on miss call `InitDefaults` → footer info (not modal).

### Byte array implementation pattern

Store data as `static constexpr std::array<juce::uint8, N>` in `.cpp` (keeps header light). Use a **one-time extraction** from fixtures during development:

```bash
# Dev-only helper — decode fixture, print C++ initializer (do not ship runtime loader)
# Matrix-Control_Tests or ad-hoc snippet with SysExDecoder
```

Commit the resulting literal bytes. Production `InitDefaults` must have **no** dependency on `MATRIX_TEST_FIXTURES_DIR`.
