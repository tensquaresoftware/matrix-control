## How to read this log

Each entry records a brownfield decision for an existing code zone:

| Decision | Meaning |
|---|---|
| **KEEP** | Keep as-is ‚Äî wire/complete only, no refactor |
| **KEEP + IMPROVE** | Keep architecture, targeted improvements (tests, perf, polish) |
| **REWIRE** | Keep UI/code, replace stub wiring (e.g. timestamps ‚Üí Core) |
| **REFACTOR** | Partial or full rework of the zone |
| **DISCARD** | Abandon (e.g. OpenGL plans, temporary dev patterns) |
| **BUILD NEW** | Does not exist yet ‚Äî specify from scratch |

### Zone 7 ‚Äî MidiManager + dedicated MIDI thread

#### D-048 ‚Äî MidiManager public API (imperative + internal queue)

- **Decision:** KEEP (public imperative API) + BUILD NEW (internal queue)
- **Rationale:** Callers (ActionDispatcher, PluginProcessor, PatchFileService) express **business intent** (`sendPatch`, `requestCurrentPatch`) ‚Äî not queue implementation detail. Public API stays ‚Äúopen book‚Äù readable; queue is dedicated-thread implementation detail.

#### D-049 ‚Äî MidiManager method naming (send vs request)

- **Decision:** KEEP + IMPROVE ‚Äî convention **`sendPatch` / `sendMaster` / `requestCurrentPatch` / `requestMaster`**
- **Rationale:** Verb = flow direction: **send** = plugin ‚Üí synth; **request** = plugin asks for SysEx response. #### D-050 ‚Äî Thread model + inbound (sync vs async)

- **Decision:** KEEP + IMPROVE ‚Äî **hybrid A** (targeted sync + async session)
- **Rationale:** **Sync (blocking request/response)** for explicit operations with known outcome: Device Inquiry at startup, `requestCurrentPatch` / `requestMaster` after user action (STORE, bank load). User accepts short wait; UI already locked without device (D-010). **Async** for the rest: `MidiManager::run()` consumes outbound queue; inbound SysEx callback (JUCE MIDI input thread) notifies MIDI thread via queue/flag ‚Äî no APVTS decode in callback. APVTS updates from inbound: `MessageManager::callAsync()` (never from MIDI or audio callback). `waitForSysExResponse` kept for RPC path; `handleIncomingSysEx` becomes async router (unsolicited responses or pending request completion). Reject **B** (all sync on MIDI thread): blocks queue during timeouts. Matrix-1000 mostly sends **responses** to our requests ‚Äî RPC model covers most of v1; async path ready for unsolicited inbound.
- **Follow-up:** Architecture startup sequence (inquiry ‚Üí unlock UI); document which calls are sync vs fire-and-forget.

#### D-051 ‚Äî Unified outbound queue (dual-role)

- **Decision:** BUILD NEW ‚Äî `MidiOutboundQueue` (or equivalent)
- **Rationale:** Brief non-negotiable: one **MIDI To** output, two producers. **Instrument** (`processBlock` / Keyboard From standalone): notes, CC, pitch bend ‚Äî **enqueue only**, filter PC and SysEx. **Editor** (message thread / ActionDispatcher): SysEx, Program Change ‚Äî enqueue. Single consumer: MIDI thread ‚Äî dequeue with **realtime priority > SysEx**, minimum delay between SysEx (`kMinSysExDelayMs`, PRD timings). Never call `MidiOutput` directly from audio thread. Header Instrument/Editor LEDs driven at enqueue/dequeue (pulse or counter ‚Äî UX detail).
- **Follow-up:** PRD rate-limit timings; queue tests under automation load; zone 8 `processBlock` enqueues instrument.

#### D-052 ‚Äî Transport class granularity (Sender / Receiver / Ports)

- **Decision:** KEEP (SRP split) + REWIRE + BUILD NEW (separate queue) ‚Äî **no** mega-class merge
- **Rationale:** **Consolidate** = group **queue/dispatch logic**, not merge Sender+Receiver+Ports into one blob. `MidiInputPort` / `MidiOutputPort` = port open wrappers (KEEP). `MidiSender` = physical write **called only from MIDI thread** (KEEP, refine). `MidiReceiver` = JUCE callback + SysEx buffer ‚Äî **REWIRE**: today `handleIncomingSysEx` in MidiManager is never called from Receiver (disconnect); Receiver must notify MidiManager (lock-free queue or registered callback). **BUILD NEW**: `MidiOutboundQueue` (+ optionally `MidiInboundDispatcher`) ‚Äî dedicated classes, MidiManager orchestrates. Reject full merge: would violate SRP, class > 200 lines, harder tests.
- **Follow-up:** Architecture component diagram; fix Receiver ‚Üí MidiManager wiring; do not call SysEx decode in Receiver.
